# Devlogs

Engineering journal for shelftxt. One file per significant change, incident, or refactor.

---

## Devlog template

Copy this block into a new file: `docs/history/devlogs/YYYY-MM-DD-short-title.md`

```markdown
# Title here

Date:
Feature / Refactor:
Problem:
Old approach:
New approach:
Files changed:
Lessons learned:
Next steps:
```

### Field guide

| Field | What to write |
|-------|----------------|
| **Date** | When the work landed (or when you wrote the log) |
| **Feature / Refactor** | One-line scope |
| **Problem** | What was broken, slow, or hard to maintain |
| **Old approach** | How things worked before |
| **New approach** | What you did instead |
| **Files changed** | Key paths (not every diff line) |
| **Lessons learned** | What you'd tell future-you |
| **Next steps** | Honest follow-ups, not a wish list |
